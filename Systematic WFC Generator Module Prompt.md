System Architecture Brief: Universal Hybrid Procedural Pipeline (UHPP)
Objective: Implement a modular JavaScript (ES2026) procedural engine. The system must move data through a multi-stage pipeline: Abstract Noise → Constraint-Based Assembly (WFC) → Iterative Simulation (CA) → Continuous Surface Reconstruction.
1. Core Tech Stack & Module Requirements
The AI should use these libraries to ensure performance and compatibility:
Logic Core: ndwfc (N-Dimensional Wave Function Collapse) for the graph-agnostic solver.
3D Engine: three (Three.js) for rendering, mesh manipulation, and GPU acceleration.
Math & Topology: d3-delaunay for Voronoi/Delaunay 3D graph generation; gl-matrix for high-performance vector/matrix operations.
Noise & L-Systems: simplex-noise for probability maps; lindenmayer for L-System string expansion.
Performance: threads.js or native Web Workers for offloading the WFC and Marching Cubes calculations.
2. The Five-Stage Pipeline Implementation
Phase A: Macro-Weighting (The "Director")
Module: NoiseHeuristic.js
Logic: Generate a 3D Simplex noise field. Map noise values to tile weights in ndwfc. (e.g., Value > 0.8 = 5x weight for "Peak" tiles).
Phase B: Skeletonization (The "Foundation")
Module: LSystemBridge.js
Logic: Use lindenmayer to generate a 3D coordinate path. Pre-collapse (pin) these coordinates in the ndwfc grid as "Path" tiles before the solver runs.
Phase C: Assembly (The "Solver")
Module: WFCManager.js
Logic:
Initialize ndwfc with a Graph generated by d3-delaunay (for Voronoi) or a TypedArray (for Cuboid).
Execute the collapse logic on a Web Worker. Use a SharedArrayBuffer to pass the grid state back to the main thread for real-time visualization.
Phase D: Iterative Simulation (The "Eroder")
Module: CellularAutomata.js
Logic: Implement a 3D CA engine that operates on the collapsed WFC grid.
Rules: "If tile is Wood and neighbor is Fire, change to Charred."
Constraint: Ensure CA never changes "Pinned" structural tiles from Phase B.
Phase E: Surface Reconstruction (The "Finisher")
Module: SurfaceSynth.js
Logic:
Convert the discrete grid into a Scalar Field (Distance Field).
Use the three/examples/jsm/objects/MarchingCubes.js implementation or metaballs logic to generate a continuous manifold mesh over the voxels.
Vertex Warping: Apply a custom ShaderMaterial in Three.js to stretch tile geometry to fit irregular Voronoi cell boundaries using Barycentric coordinates.
3. Technical Specification for Code AI
Memory Management: Use TypedArrays (Uint8Array, Float32Array) for all voxel and graph data to minimize Garbage Collection.
Concurrency: Implement an AsyncManager that handles the hand-off between the Solver (Worker) and the Renderer (Main Thread).
API Structure: Provide a Pipeline class with methods .addStage(stage) and .execute(), ensuring the output of one phase (e.g., Tile IDs) is the valid input for the next (e.g., CA Grid).
